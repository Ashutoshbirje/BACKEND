# Component of API :

1) Endpoint :  It is access point to send the HTTP request to and get your response

2) Paths : [URL --> Endpoint/Paths]

3) Parameters : [URL --> Endpoint/Paths?Parameter=value]

4) Query : [URL --> Endpoint/Paths?(q1&q2&(Parameter=value)&...)]

4) Authetication

# REST API (Representational State Transfer)

--> Architectural Style to build API 
--> Use Specific Pattern of Routes / Endpoint URLs
--> Use HTTP Request Verbs 

             HTTP request
              FTP request
             HTTPS request
--> [Client] <-------------> [Server] <-------------> [Database]
                 [API]

# HTTP verbs 

1) GET (READ)
2) POST (CREATE)
3) PUT (UPDATE) : Update database with new entry 
4) PATCH (UPDATE) : Update database with modification
5) DELETE (DELETE)
